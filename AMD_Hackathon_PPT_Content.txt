  - Product search mode runs full LLM review analysis on top candidates before ranking
  - Ranking itself is local Python (no tokens); DuckDuckGo search is external (no LLM tokens)

One-liner for slide:

GPU USAGE & MEMORY (RIGHT-SIZING)
---------------------------------
Hardware: AMD Instinct MI300X — 192 GB VRAM

Single GPU deployment (typical hackathon pod):
  Text (Qwen3-32B):   --gpu-memory-utilization 0.55 (~105 GB)
  Vision (Qwen2.5-VL-7B): 0.18 (~35 GB) on same GPU
  Total VRAM: ~130–145 GB / 192 GB (~68–75%)

Dual GPU deployment (if 2× MI300X available):
  GPU 0 — Text @ 85% (~163 GB)
  GPU 1 — Vision @ 30% (~58 GB)
  Split across 2 GPUs

Why MI300X fits this workload:
  - Runs 32B text + 7B vision concurrently on one 192 GB card
  - bf16 inference via vLLM on ROCm
  - Large VRAM avoids model offloading and supports long JSON outfit responses (up to 4K tokens)

SLIDE 2: IMPACT & DEMO SUMMARY
================================================================

EXPECTED IMPACT / VALUE
-----------------------

For shoppers (experience):
  - Turns a vague request ("I'm going to a wedding, budget ₹15K") into 3 shoppable
    outfit boards with real Amazon buy links
  - Personalizes via photo analysis (skin tone, body type, color palette) when user
    uploads a photo
  - Surfaces ranked products with review-aware scoring — not just search results

For business (efficiency & scale):
  - Agentic pipeline replaces manual browsing across Amazon, Flipkart, Myntra, Ajio, Nykaa
  - Local LLM inference on AMD — no per-query cloud API cost, data stays on-prem
  - Modular tools (vision → stylist → search → review → rank) scale to other domains

Productivity gain (demo narrative):
  "What takes 30–45 minutes of manual comparison across sites → under 1 minute with
   ranked, explained recommendations."


KEY DIFFERENTIATORS / INNOVATION
--------------------------------
1. Multi-agent orchestration on AMD — single Gradio UI routes to vision, stylist,
   shopping, and outfit agents via intent classification

2. Dual-model architecture on MI300X — Qwen3-32B (reasoning) + Qwen2.5-VL-7B (vision)
   co-located via vLLM + ROCm

3. Review intelligence, not just ratings — LLM extracts pros/cons/sentiment; composite
   ranking: rating 40% · review quality 30% · budget 20% · style 10%

4. Complete outfit assembly — generates 3 distinct occasion-based looks, searches each
   slot (top/bottom/footwear/accessories), gender-aware filtering

5. India-first commerce — DuckDuckGo + site:amazon.in / Flipkart / Myntra with INR
   budget parsing ("15k", "₹15000")

6. Demo-hardened — fast outfit mode, fallback outfit templates, resilient Gradio UI,
   vision timeout guards

ARCHITECTURE (OPTIONAL EXTRA SLIDE)
================================================================

User → Orchestrator → [Vision Tool] → Stylist Tool → Product Search → Review Analysis → Ranking → Response

Pipeline modules:
  1.  Environment Setup       — pip install, ROCm GPU check
  2.  vLLM Clients            — text_client, vision_client, llm_json_call()
  3.  Data Models             — UserProfile, Outfit, Product, Recommendation, etc.
  4.  Vision Tool             — analyze_user_image() → Qwen2.5-VL
  5.  Stylist Tool            — generate_outfits() → Qwen3-32B
  6.  Product Search          — search_products() → DuckDuckGo + site: filters
  7.  Review Analysis         — analyze_reviews() → Qwen3-32B
  8.  Ranking                 — rank_products() → weighted scoring
  9.  Shopping Agent          — recommend_products() → search + review + rank
  10. Outfit Agent            — build_complete_outfits() → stylist + search + assemble
  11. Orchestrator            — StylistOrchestrator.process() → intent routing
  12. Gradio UI               — Chat + image upload + HTML product/outfit cards
